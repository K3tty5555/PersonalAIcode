# 竞品研究

## 执行协议

- 基于产品类型识别相关竞品
- 进行功能、体验、商业模式的对比分析
- **用户确认**：询问用户是否有特别关注的竞品
- 识别差异化机会点
- 输出结构化的竞品分析报告

## 研究框架

### 0. 开始前询问用户补充

**阶段启动询问：**
```
🔄 即将开始：竞品研究

基于需求分析，我计划研究以下方向的竞品：
• 直接竞品：解决同类问题的产品
• 间接竞品：解决方式不同的替代方案
• 参考竞品：跨领域可借鉴的产品

💬 在开始研究前，请问：
   你知道哪些竞品或类似产品需要特别关注？

例如：
• 你们现在用的是什么系统/产品？
• 你觉得哪个竞品做得好？为什么？
• 哪个竞品做得差？有什么坑要避免？
• 有什么跨领域的优秀产品值得参考？

请直接告诉我竞品信息，或回复"没有补充，按你的分析来"。
```

**处理用户补充：**
- 如果用户提及具体竞品 → 重点分析用户提到的竞品
- 如果用户有评价 → 记录用户评价，作为分析参考
- 如果用户无补充 → 基于需求自动识别竞品

---

### 1. 竞品识别与用户确认

**自动识别：**
- **直接竞品**：解决相同问题、面向相同用户
- **间接竞品**：解决相同问题、方式不同
- **参考竞品**：不同领域但有可借鉴之处

**用户确认（输出初稿前）：**
```
我分析了以下竞品：

直接竞品：{列出识别的竞品}
间接竞品：{列出识别的竞品}

请问：
1. 有没有我特别关注的竞品需要深入分析？
2. 你知道哪些我没提到的竞品？
3. 你对这个市场的了解有哪些？

（如果没有特别补充，我就基于以上分析生成报告）
```

### 2. 分析维度

**功能维度：**
- 核心功能对比
- 功能完整度
- 特色功能

**体验维度：**
- 交互设计
- 视觉风格
- 易用性

**商业维度：**
- 定价策略
- 盈利模式
- 用户规模

**技术维度：**
- 技术路线
- 性能表现
- 创新点

### 3. SWOT 分析

对每个主要竞品进行 SWOT 分析。

### 4. 差异化机会

基于竞品分析，识别：
- 市场空白点
- 竞品弱点可突破
- 独特优势可发挥
